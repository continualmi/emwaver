Flashing guide (docs)
https://luispl77.github.io/emwaver/flashing-firmware/esp32/

This video walks through building and flashing custom firmware onto ESP32-S3 EMWaver devices (Flagship, Shield, DIY, and ESP32-S3 dev boards).

You’ll learn
- Generate a firmware project with emwaver-cli
- Install ESP-IDF and build with idf.py
- Flash over USB on Linux/macOS
- Flash on Windows using WSL2 + usbipd-win
- Verify the firmware over BLE with emwaver shell

ESP-IDF (ESP32-S3 “Get Started”)
https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/get-started/

Install emwaver-cli
curl -fsSL https://raw.githubusercontent.com/luispl77/emwaver/main/cli/install.sh | sh

Create a firmware project
mkdir -p ~/projects/emwaver-firmware && cd ~/projects/emwaver-firmware
emwaver init --target esp32s3 --path .
source setup.sh

Build + flash (Linux/macOS/WSL)
idf.py set-target esp32s3
idf.py build
idf.py -p PORT flash
idf.py -p PORT monitor

Tip: on Linux the port is often /dev/ttyUSB0 or /dev/ttyACM0. On macOS it’s usually /dev/cu.*

Verify over BLE (requires Bluetooth enabled)
emwaver shell

macOS note (CH34x USB-to-serial)
If your board shows up as a CH34x device, install the driver:
brew install --cask wch-ch34x-usb-serial-driver
https://formulae.brew.sh/cask/wch-ch34x-usb-serial-driver

Windows note (WSL2 + usbipd-win)
Attach the USB device to WSL (PowerShell as Administrator):
winget install dorssel.usbipd-win
usbipd list
usbipd bind --busid BUSID
usbipd attach --busid BUSID --wsl

If you get permission errors in WSL when flashing:
sudo usermod -a -G dialout $USER
newgrp dialout
