# `.emwaver/`

This folder is the repo-local home for “vibe hacking” runbooks and agent skills.

Constraints:
- **Markdown-only**: keep files as `.md` so they are easy to review, diff, and feed into tooling.
- Treat this as the source of truth for pinmaps, known-good command snippets, and module bring-up recipes.

Contents (suggested):
- `SPI.md`: proven SPI open/xfer recipes and register maps.
- `WAVELETS.md`: how to turn a working probe into a Wavelet UI.
- `skills/`: small, task-oriented “skills” in Markdown that agents can follow.
