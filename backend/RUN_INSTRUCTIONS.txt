EMWaver Backend - Run Instructions

Local dev (recommended)

1) Create venv + install deps:

  cd backend
  python3 -m venv .venv
  source .venv/bin/activate
  pip install -r ../requirements.txt

  # Optional: Postgres driver (only if you set DATABASE_URL to Postgres)
  pip install -r ../requirements-postgres.txt

2) Configure env:

  # Auth is always enforced (Firebase ID tokens)
  export FIREBASE_PROJECT_ID=emwaver-64527

  # Optional: CORS
  export CORS_ORIGINS=*

  # Optional: DB (defaults to sqlite:///./emwaver.db)
  export DATABASE_URL=sqlite:///./emwaver.db

  # Agent chat (required if you call /api/agent/chat)
  export OPENROUTER_API_KEY=...

3) Start:

  python app.py

4) Smoke tests:

  curl http://127.0.0.1:8787/health

  # In dev auth-bypass mode, this should return an empty list initially.
  curl "http://127.0.0.1:8787/v1/files?kind=script&ext=.emw&include_content=0"


Auth (Firebase)

- Backend expects:
  Authorization: Bearer <firebase_id_token>

- Required env:
  export FIREBASE_PROJECT_ID=...  # Firebase project id


Docker

  # Build from repo root so Docker can access repo-wide requirements.txt
  docker build -f backend/Dockerfile -t emwaver-backend .

  docker run --rm -p 8787:8787 \
    -e FIREBASE_PROJECT_ID=... \
    -e CORS_ORIGINS='*' \
    emwaver-backend

  # With agent chat enabled
  docker run --rm -p 8787:8787 \
    -e FIREBASE_PROJECT_ID=... \
    -e OPENROUTER_API_KEY=... \
    emwaver-backend


Azure Postgres (later)

- Point DATABASE_URL at your Azure Postgres instance:
  export DATABASE_URL='postgresql+psycopg2://USER:PASSWORD@HOST:5432/DBNAME?sslmode=require'
